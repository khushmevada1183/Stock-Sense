You are a senior product designer and frontend engineer. Redesign the entire Stock Sense website into a premium, Apple-inspired financial product experience. Use the redesigned portfolio page as the benchmark and extend that same visual language across every page, shared component, and UI state. The current interface should be treated as legacy. Rebuild it so the whole website feels modern, calm, technical, high-trust, and expensive in both light mode and dark mode.

Start with a full audit of the app. Identify all shared components, page templates, spacing patterns, typography styles, cards, tables, buttons, forms, charts, modals, empty states, navigation, and footers. Do not redesign one page in isolation. First build a unified design system, then apply it consistently across the whole site.

Design direction:
- Premium, minimal, and restrained.
- Apple-like in clarity, spacing, and polish, but tailored for a stock market product.
- No clutter, no noisy gradients, no unnecessary decoration, no generic admin-dashboard look.
- Use soft depth, clean surfaces, strong hierarchy, and elegant spacing.
- Light mode should feel airy, refined, and premium.
- Dark mode should feel deep, elegant, and technically confident.
- Use a limited color system with emerald green as the main accent and subtle supporting tones.
- Make every visual choice feel intentional.

Typography and layout:
- Use a strong hierarchy with clear section titles, clean labels, and prominent numeric values.
- Remove redundant text, filler copy, and repeated hints.
- Prefer spacious layouts with balanced grids and well-controlled density.
- Make tables, metrics, charts, and forms feel like a premium product, not a generic dashboard.
- Keep layouts responsive and polished on mobile, tablet, and desktop.
- Use consistent spacing, radius, shadows, and border treatment across the entire product.

Motion and interaction:
- Add subtle, purposeful motion only where it improves clarity and feel.
- Use smooth fades, gentle lifts, refined hover states, and restrained scroll reveals.
- Avoid flashy animation, gimmicks, or motion that distracts from the data.
- Respect reduced-motion preferences.
- Keep interactions crisp, stable, and fast.

Implementation rules:
- Refactor repeated UI into reusable components and shared design tokens.
- Preserve all product functionality, routes, API behavior, and business logic unless a UI change truly requires a safe adjustment.
- Improve accessibility: semantic structure, focus states, keyboard support, and contrast.
- Apply the new language to every major area: home, stocks, market, IPOs, news, portfolio, watchlists, auth screens, forms, dialogs, filters, empty states, tables, charts, and shared shell components.
- If a component feels old, simplify it, restructure it, or replace it rather than layering more styling on top.

Working style:
- First audit the current UI system.
- Then define the new visual language and component rules.
- Then implement the redesign page by page using reusable components.
- Keep the portfolio page as the quality reference for premium feel, spacing, and restraint.
- When in doubt, choose clarity, balance, and restraint over decoration.
